1 DockerCompose

1.1 简介

参考官方文档：https://docs.docker.com/compose/

Compose is a tool for defining and running multi-container Docker applications. With Compose, you use a YAML file to configure your application’s services. Then, with a single command, you create and start all the services from your configuration. To learn more about all the features of Compose, see the list of features.

  1. 定义运行多个容器
  2. YAML 配置文件 docker-compose.yml
  3. 使用命令创建并启动所有服务

Compose works in all environments: production, staging, development, testing, as well as CI workflows. You can learn more about each case in Common Use Cases.

Using Compose is basically a three-step process:
三步骤：

  1. Define your app’s environment with a Dockerfile so it can be reproduced anywhere.

    • 定义 dockerfile 文件，保证我们的项目在任何地方可以运行。）

  2. Define the services that make up your app in docker-compose.yml so they can be run together in an isolated environment.

    • service：什么是服务？
    • docker-compose.yml 文件怎么写？

  3. Run docker compose up and the Docker compose command starts and runs your entire app. You can alternatively run docker-compose up using the docker-compose binary.

    • 启动项目

作用： 批量容器编排

理解

Compose 是 Docker 官方的开源项目，需要安装！

Dockerfile 让程序运行在任何地方。

docker-compose.yml 文件的内容：

docker compose up 100 多个服务。

Compose 重要概念：

  • 服务 Service：容器、应用，web、redis、mysql、nginx……
  • 项目 project：一组关联的容器，一个博客可能会包含：web、redis、mysql、nginx……多个容器。

2 Docker Swarm

参考官方链接：https://docs.docker.com/engine/swarm/how-swarm-mode-works/nodes/

2.1 准备工作

搭建集群环境：

  1. 需要在阿里云上买服务器，一些配置信息。（登录阿里云–>控制台–>云服务器 ECS–>创建实例）

    • 付费方式：按量付费
    • 地域及可用区：就近选择
    • x86 计算，共享型，1 核 2 G，4 台
    • 公共镜像 Centos
    • 存储：云盘 选上随实例释放
    • 网络：选择默认的安全组。（一定要在同一个安全组内，否则 ping 不通）
    • 带宽：可以拉高点（反正是按量付费）
    • 登录凭证：选择自定义密码，设置登录密码

  2. 使用 XShell 连接 4 台服务器

  3. 在 4 台服务器上安装 Docker，XShell 可以鼠标右键设置“发送键输入到所有会话”，同步操作

2.4 Raft 协议

集群的模式：

  • 双主双从：当一个 manager 节点（该节点作为 Leader）挂了，另外一个 manager 节点（作为 Reachable）也不能用了！
  • 三主一从：当一个 manager 节点挂了，其他 manager 节点依然可用。

Raft 协议：保证大多数节点存活才可以使用，只要 manager 大于 1 台就可以，但是集群节点至少大于 3 台

注：worker 节点就是工作的，一些命令只能在 manager 节点上进行操作。

2.5 体会

弹性、扩缩容、集群

docker run 启动一个容器 ==> docker-compose up 单机启动一个项目 ==> docker service （swarm 集群）

容器 ==> 服务 ==> 副本 （例如：redis 服务 ==> 10 个副本（也就是开启 10 个 Redis 容器））

docker service 相关命令：创建服务（create）、动态扩展服务（scale）、动态更新服务（update）

灰度发布：金丝雀发布。（发布不影响网站的使用）

  1. 启动服务
  2. 查看服务
  3. 动态扩缩容
  1. 移除服务

服务：集群中任意节点都可以访问，可以有多个副本动态扩缩容实现高可用！

docker swarm 并不难：只要会搭建集群、会启动服务、动态管理容器就可以！swarm 是简单的 K8S

2.6 概念总结

swarm: 集群的管理和编号。docker 可以初始化一个 swarm 集群，其他节点可以加入（manager、worker）
Node: 就是一个 docker 节点，多个节点组成了一个网络集群（manager、worker）
Service: 任务，可以在管理节点（manager）或者工作节点（worker）来运行。核心！用户访问！
Task: 容器内的命令，细节任务。


执行 docker service create 命令后的流程：
命令–>管理(manager 节点)–>api–>调度–>工作节点（创建 Task 容器维护创建）
