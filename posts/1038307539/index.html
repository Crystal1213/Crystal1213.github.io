Linux 系统管理

发表于 2021-03-25 更新于 2021-03-29 分类于 学习笔记 阅读次数：
本文字数： 6k

Linux 系统管理方面的知识，重点讲解：

  1. 进程相关的知识，以及查看系统的进程相关的命令。
  2. 工作管理相关的内容，例如：如何将进程放入后台执行等。
  3. 日志管理相关内容。

1 系统运行级别

1.1 运行级别

Linux 默认有七个运行级别：

  • 0：关机
  • 1：单用户模式，可以想象为 windows 的安全模式，主要用于系统修复
  • 2：不完全的命令行模式，不含 NFS 服务（Linux 系统之间文件传输服务）
  • 3：完全的命令行模式，就是标准的字符界面
  • 4：系统保留模式
  • 5：图形模式
  • 6：重启动

查看系统运行级别的命令：

  • runlevel：结果显示 N 3 （N 表示这个级别前，上一个是哪个级别，3 代表当前级别）

改变当前运行级别的命令：

  • init 级别：例如init 5切换成图形模式，当然系统首先要安装图形界面才可以
  • 说明：init 0 虽然可以关机，init 6 可以重新启动，但是不建议使用，容易造成数据丢失

1.2 系统默认运行级别

系统的默认运行级别是写在配置文件 /etc/inittab 中，但是 RedHat 6 以后该配置文件就不再生效

1.3 /etc/rc.d/rc.local 文件

/etc/rc.d/rc.local 这个文件会在用户登录之前读取，文件中写入什么命令，系统每次启动时都会执行一次。如果我们想让系统在启动时做什么工作，只需写入 /etc/rc.d/rc.local 这个配置文件即可。

默认情况下，/etc/rc.d/rc.local 文件中会有命令 touch /var/lock/subsys/local ，可通过查看该文件的时间参数来判断系统启动的时间。

2 启动引导程序（Boot Loader）

引导程序就是引导系统的启动过程，Lilo 引导程序已经不是很常见了，现在大多是 grub 引导程序，主要有以下优势：

  • 支持更多的文件系统
  • grub 主程序可以直接在文件系统中查找内核文件
  • 在系统启动时，可以使用 grub 的交互界面编辑和修改启动选项
  • 可以动态的修改 grub 的配置文件，在修改配置文件之后不需要重新安装 grub，只需要重新启动就可以生效

2.1 /boot/grub 目录

grub 的作用：

  • 加载操作系统的内核
  • 拥有一个可以让用户选择的菜单，来选择到底启动哪个系统
  • 调用其它的启动引导程序，来实现多系统引导

grub 的配置文放在 /boot/grub 目录中

3 服务

3.1 服务的简介与分类

Linux 服务的分类：

  • RPM 包默认安装的服务（这类服务可以被服务管理命令识别）
    • 独立的服务：独立启动，不依赖其他的管理服务，当客户端请求访问时，独立的服务请求更快。例如：apache 服务、FTP 服务、Samba 服务。
    • 基于 xinetd 服务：不能独立启动，xinetd 服务是系统的超级守护进程，其作用就是管理不能独立启动的服务，当客户端请求访问时，先请求 xinetd 服务，由 xinetd 服务去唤醒相对应的服务。（xinetd 服务是要自己安装的，yum -y install xinetd）
  • 源码包安装的服务

查询已安装的服务和的命令：chkconfig --list

4 RPM 包默认安装的服务管理

4.1 独立服务管理

服务的启动方法：

  • 使用 /etc/init.d/ 目录中的启动脚本启动服务，执行命令 /etc/init.d/httpd 服务名，例如：/etc/init.d/httpd start （推荐使用这种方式，因为下面这种方式无法开启按照源码包的方式安装的服务）！！！
  • 使用 service 命令来启动独立的服务，执行命令 service 服务名 start|stop|restart|...，例如：service httpd start
  • 但是在 Centos 7 中有些不同，上述第一种启动方式已经失效，第二种启动方式虽然能用，但是会映射到另外一个命令 systemctl，因此可以使用 systemctl start httpd

服务的自启动方法：

  • 使用 chkconfig 命令，执行命令 chkconfig [--level 运行级别] [独立的服务名] [on|off]，例如：chkconfig --level 2345 httpd on ，但是在 centos 7 中 chkconfig 命令已经替换成了 systemctl
  • 修改 /etc/rc.d/rc.local 文件，设置服务自启动，加入服务启动命令，例如：/etc/rc.d/init.d start （Centos 7 中依然不适用）
  • 使用 ntsysv [--level 运行级别]，会进入图形界面选择要启动的服务。

5 进程（这部分内容较重要哦）

5.1 进程的查看

ps 命令：（进程的状态静态显示）

  • 查看系统中所有进程，使用 BSD 操作系统格式：ps aux 注意这个命令的 aux 前没有 - ，a 表示显示一个终端的所有进程，除了会话引线，u 表示显示进程的归属用户及内存的使用情况，x 显示没有控制终端的进程
  • 查看系统中所有进程，使用 Linux 标准命令格式：ps -le ，l 表示长格式显示更加详细的信息，e 表示显示所有的进程，和 A 的作用一样

top 命令：（进程的状态动态显示）

  • top [选项]

  • 选项：

    • -d 秒数：指定 top 命令每隔几秒更新，默认是 3 秒
    • -b：批处理模式输出，一般和 -n 选项一起使用，用于把 top 命令重定向到文件中
    • -n 次数：指定 top 命令执行的次数，一般和 -b 选项一起使用
    • -p：指定 PID。只查看某个 PID 进程
    • -u 用户名：只监听某个用户的进程

  • 在 top 命令的交互模式当中可以执行的命令：

    • P：以 CPU 的使用率排序，默认就是此项
    • M：以内存的使用率排序，
    • N：以 PID 排序
    • T：按照 CPU 累计运算时间排序

以进程树的形式显示系统中的进程：pstree [选项]

选项：

  • -p：显示进程的 PID
  • -u：显示进程的所属用户

5.2 进程管理

查看系统中可以识别的信号的命令，kill -l 或者 man 7 signal

重点说下 SIGKILL 信号，该信号代号是 9，其作用是立即结束程序的运行，一般用于强制终止进程。

  • kill 命令：kill [信号] PID
  • killall 命令：kill [选项][信号] 进程名
    • -i 选项：交互式，询问是否要杀死某个进程
    • -I 选项：忽略进程名的大小写
  • pkill 命令：pkill [信号] [选项] 进程名
    • -t 终端号：按照终端号踢出用户，举例pkill -9 -t pts/1

6 工作管理

6.1 工作管理简介

  • 前台是指当前可以操控和执行命令的这个操作环境，后台是指工作可以自行运行，但是不能直接用 ctrl+c 来终止它，只能使用 fg/bg 来调用工作；
  • 当前的登录终端，只能管理当前终端的工作，而不能管理其它登录终端的工作。例如：tty1 登录的终端不能管理 tty2 终端中的工作；
  • 放入后台的命令必须可以持续执行一段时间，这样才能捕捉和操作这个工作。如果把 ls 命令放入后台执行，它很快就会执行完成，我们很难操作它；
  • 放入后台执行的命令不能和前台用户有交互或需要前台输入，否则放入后台只能暂停，而不能执行。比如 vi 命令放入后台只能暂停，而不能执行，因为 vi 需要前台输入信息。top 命令也不能放入后台执行，而只能放入后台暂停，因为 top 命令需要和前台有交互。

6.2 将命令放入后台

方法：

  • 命令 &：使用该方法放入后台是执行状态，例如 find / -name install.log &
  • 命令执行过程中，按下 ctrl + z：使用该方法放入后台命令是暂停状态，例如执行 top 命令然后按下 ctrl + z

6.3 后台管理命令

  • 查看后台的工作：jobs [-l] 其中 -l 选项表示显示工作的 PID
  • 将后台暂停的工作恢复到前台执行：fg %工作号 其中 % 可以省略。（说明：工作号不同于 PID 号，工作号是进程放入后台的顺序）
  • 将后台暂停的工作恢复到后台执行：bg %工作号
  • 后台命令脱离登录终端运行：（因为命令放入后台，只能在当前登录终端执行）
    • 把需要后台执行的命令加入到 /etc/rc.local 文件，让系统启动时执行这个后台程序。（这种方法要求服务器必须重启，因为只有重启服务器时，才会执行这个文件中的命令。）
    • 使用定时任务
    • 使用 nohup [命令] & 命令，让后台工作在离开操作终端时，也能够在后台执行。例如：nohu find / -print > /root/file.log &

7 系统资源查看

7.1 vmstat 命令监控系统资源

  • 格式：vmstat [刷新延时 刷新次数] 例如 vmstat 1 3

7.2 dmesg 显示开机时内核检测信息

举例：dmesg | grep CPU （显示系统开机时，CPU相关的信息）

7.3 free 命令查看内存使用状态

  • 格式：free [-b|-k|-m|-g]
  • 选项：
  • -b：以字节为单位显示
  • -k：以 KB 为单位显示，默认就是以 KB 为单位
  • -m：以 MB 为单位显示
  • -g：以 GB 为单位显示

7.4 查看 CPU 信息

CPU 主要信息保存在 /proc/cpuinfo 这个文件夹中，执行命令 cat /proc/cpuinfo 就可看到 CPU 相关信息

7.5 查看本机登录用户信息

执行命令 w 或者 who

作用：显示系统启动时间与平均负载

7.7 查看系统与内核相关信息

  • 命令格式：uname [选项]
  • 选项：
    • -a：查看系统所有相关信息
    • -r：查看内核版本
    • -s：查看内核名称
  • 查看系统位数的命令：可通过 file 命令来判断系统文件（主要是系统命令）的位数，例如 file /bin/ls
  • 查看当前 Linux 系统的发行版本：执行命令 lsb_release -a （执行该命令前，需要安装该命令才行，执行命令 yum -y install redhat-lsb）

8 系统定时任务

8.1 at 一次性执行定时任务

  • at 命令要正确执行需要 atd 服务的支持（一般服务后面都带 d 选项）
  • at 访问控制是依靠文件 /etc/at.allow 文件（白名单）和 /etc/at.deny 文件（黑名单）这两个文件实现的，但是系统中默认只有 /etc/at.deny 文件（黑名单）。原因在于：只有加入白名单的用户才能使用 at 命令，如果有一个用户加入了白名单，其他用户不加入就不能使用。
  • 默认情况下 /etc/at.deny 文件是空的，这样系统中的所有用户都能使用 at 命令
  • /etc/at.allow 文件的权限更高，如果 /etc/at.deny 文件存在，/etc/at.allow 文件就失效。而且这两个文件对 root 用户都无效。

at 命令：

  • 命令格式：at [选项] 时间

  • 选项：

    • -m：当 at 工作完成后，无论是否命令有输出，都用 email 通知执行 at 命令的用户
    • at -c 工作号：显示该 at 工作的实际内容，举例：at -c 8

  • 时间：（时间格式）

    • HH:MM：在指定的“小时：分钟”执行命令，例如：02:30
    • HH:MM YYYY-MM-DD：在指定的“小时:分钟 年-月-日”执行命令，例如：02:30 2018-07-25
    • HH:MM[am|pm] [mounth] [date]：在指定的“小时:分钟[上午|下午][月][日]”执行，例如：02:30 July 25
    • HH:MM[am|pm]+[minutes|hours|days|weeks]：在指定的时间“再加上多久执行”，例如：now+5minutes，05am+2hours

  • 举例：键入命令 at now+2minutes 回车键后输入要执行的命令，输入完后 ctrl+d 保存

  • 查询当前服务器上的 at 工作：atq

  • 删除当前服务器上的 at 工作：atrm [工作号]

8.2 crountab 循环执行定时任务

crountab 命令是需要 cround 服务支持的，和 at 命令类似，也是通过 /etc/cron.allow 和 /etc/cron.deny 文件来限制某些用户是否可以使用 crontab 命令。

  1. 使用 crontab -e 来设置定时任务
    crontab -e 来设置定时任务，默认是当前登录用户。
  • 命令格式：crontab [选项]
  • 选项：
    • -e：编辑 crontab 定时任务
    • -l：查询 crontab 任务
    • -r：删除当前用户的所有 crontab 任务
    • -u 用户名：修改或删除其他用户的 crontab 任务，只有 root 可用
  • 说明：执行 crontab -e 命令后，会进入编辑界面，编辑定时任务。

编辑定时任务的基本格式：* * * * * 执行的任务

  1. 将定时任务写入配置文件 /etc/crontab
    修改配置文件 /etc/crontab 来设置定时任务，并不是说不需要用户身份，而是定时任务的执行者身份可以手工指定。

两种方法：

  • 将定时任务写成脚本程序，并赋予执行权限，然后直接把这个脚本复制到 /etc/cron.{hourly,daily,weekly,monthly} 目录中的任意一个（例如：每周执行就复制到 weekly）
  • 修改 /etc/crontab 这个配置文件，自己加入自己的定时任务，不过要注意指定脚本的执行者身份

8.3 anacron

anacron 的作用：如果定时任务在需要执行的时间点，恰好服务器没有开机，那么这个定时任务就不会执行，当服务器开机后，会有 anacron 来处理处理定时任务的执行。

9 日志管理

9.1 系统中常见的日志文件

  • /var/log/cron：记录定时任务相关的日志
  • /var/log/cups/：记录打印信息的日志
  • /var/log/dmesg：记录了系统在开机时内核自检的信息。也可以使用 dmesg 命令直接查看内核自检信息
  • /var/log/btmp：记录错误登录的日志，该文件需要用 lastb 命令查看
  • /var/log/lastlog：记录系统中所有用户最后一次登录时间的日志，需要使用 lastlog 命令
  • /var/log/maillog：记录邮件信息
  • /var/log/messages：记录系统重要信息的日志。这个日志文件中会记录 Linux 系统中绝大多数重要信息，系统出现了问题首先要查看这个日志
  • /var/log/secure：记录验证和授权方面的信息，只要涉及账户和密码的程序都会记录。例如 ssh登录，su 切换账户，sudo 授权，以及添加用户和修改用户都在这个文件中
  • /var/log/wtmp：永久记录所有用户的登录、注销信息，同时记录系统的启动、重启、关机事件，需要使用 last 命令来查看
  • /var/run/utmp：记录当前已经登录的用户信息，这个文件需要使用 w、who、users 命令来查询
  • 采用 RPM 方式安装的系统服务也会默认把日志记录在 /var/log/ 目录中（源码包安装的服务日志是在源码包指定目录中）
